Hướng dẫn tạo web tĩnh - trang cá nhân trên Github

2019-02-28 Nguyễn Hồng Linh

Nói đến trang web các nhân, mọi người hay nghĩ đến các trang blog cá nhân, và nghĩ đến các Opensource CMS (Content Managenment System, hệ thống quản lý và xuất bản bài viết có mã nguồn mở)

CMS thường sẽ gồm 2 phần, phần Admin để bạn quản lý cơ sở dữ liệu backend (các bài viết, comment, media, ảnh…) và phần frontend là trang clent (trang chủ) được publish (đăng lên) các nội dung bài viết từ admin cho người dùng đọc.

Điểm chung của CMS đó là ta phải thuê một Hosting để lưu trữ mã nguồn và cơ sở dữ liệu của trang web. Như thế thì một trang web miễn phí (vì lấy từ opensource) sẽ không còn miễn phí nữa. CHúng ta phải đóng phí duy trì hosting trang web hàng tháng để web của ta không “chết”. Ngoài Hosting ra, ta còn phải mua thêm Domain (tên miền) cho trang web để trỏ đến hosting (vốn là 1 địa chỉ ip khó nhớ). Vậy, để làm ra 1 website cá nhân ta thường có công thức sau:
>**Download mã nguồn Opensource về + Tải theme + Chỉnh sửa và viết nội dung và Database + Mua Hosting + Mua tên miền + Upload code và database lên Host = Trang web cá nhân.

Trang cá nhân là trang như thế nào?

Trang web các nhân thường là các trang giới thiệu về bản thân (Profile cá nhân) hoặc một sạng Blog mà bạn chia sẻ các bài viết mà bạn viết hoặc sưu tầm được. Cũng có thể là một trang giới thiệu sản phẩm mà bạn làm ra.

Trang web cá nhân thường là trang tĩnh. Nghĩa là chỉ có một người dùng, không có chức năng đăng nhập và thêm dữ liệu từ người khác ngoài bạn, người dùng chỉ có đọc tin, và tương tác duy nhất người dùng có thể làm là viết comment cho bài viết.
Trang web tĩnh cũng có thể là trang báo điện tử Lúc này trang của bạn chỉ xuất bản thông tin đến người đọc. Người đọc có thể comment cho các bài viết như trang Blog.

Một trang web tĩnh cũng có thể là Landing Page (Trang web chỉ có 1 trang duy nhất) dùng để giới thiệu về công ty hay một sản phẩm nào đó.

Trang tĩnh không cần đến cơ sở dữ liệu để lưu trữ, chỉ cần một bộ giao diện gồm Html, ít Css và Javascript. Trang tĩnh sử dụng một bộ Template (bộ khung) trang web, tác các phần lặp lại riêng thành một phần, phần ruột của trang viết thành một phần và khi chạy sẽ ghép lại với nhau. Để làm được điều này ta cần một “Web Static Generator” (Bộ công cụ tạo trang web tĩnh từ template).
Cách hoạt động của công cụ tạo trang tĩnh (Static Site Generator) như sau:

Điểm mạnh của Static Site Generator (SSG) so với HTML thuần

SSG có thể tạo ra website đa ngôn ngữ rất nhanh và mạnh
Chúng ta cần một trang tĩnh nhưng lại đa ngôn ngữ, ví dụ Blog 2 ngôn ngữ Anh-Việt, hoặc web laning page giới thiệu sản phẩm nhiều ngôn ngữ khác nhau, Hugo hoặc các công cụ SSG khác có thể giúp bạn.
SSG hỗ trợ tạo ra một Blog chuẩn
Nếu chúng ta cần một trang web cá nhân, trang đầu tiên chúng ta muốn làm hiển nhiên là một trang blog, trang để viết bài và hiển thị theo ngày tháng như một tờ nhật ký hay tờ báo. Lúc này chúng ta cần các chức năng cơ bản sau đay của một trang Blog:

  1. Hiển thị bài viết (post) theo ngày.
  2. Lọc theo ngày tháng.
  3. Hiển thị nhóm bài viết theo thẻ (tags), hoặc mỗi bài gắn cho nó một danh mục (category) để gom được các bài viết liên quan lại với nhau.
  4. Tìm kiếm được bài viết theo tiêu đề hoặc nội dung bên trong của toàn bộ Blog.
  5. Hỗ trợ soạn thảo và hiển thị các embeded (nội dung của bên thứu ba, vd: Youtube, Slideshare…) trực tiếp trong bài viết.
  6. Cho phép bình luận dưới mỗi bài viết.
  7. Hiển thị các bài viết liên quan đến bài đang đọc.
  8. Hỗ trợ SEO mạnh để làm tăng view.
  9. Cần có Goole Analytic để xem người xem đến từ đâu và làm SEO cho chuẩn.
  10. Có RSS để bắt tay được với các trình đọc tin tức ngắn gọn (kiểu feedly, google news) nữa thì tốt.
  11. Chia sẻ được các bài viết sang mạng xã hội.

Các điểm trên đây Hugo làm ngon. Và hoàn toàn miễn phí, trang bị tận răng ngay khi cài đặt không cần bất kỳ Plugin nào.

SSG tạo giao diện rất nhanh và dễ chỉnh sửa
Bởi vì giao diện được viết dưới dạng html đính kèm data.
SSG cho phép nội dung được lưu trữ dưới dạng file, không cần cơ sở dữ liệu
Khi làm web động, chúng ta hiển nhiên đi tìm một cơ sở dữ liệu để lưu trữ data của website như MySQL, MSSQL, MonggoDB… nhưng làm web dạng SSG thì ta lưu trữ data vào file và dữ liệu hoàn toàn dạng text-base (thường là JSON hoặc TOML). Vì dữ liệu là dạng tree như JSON nên chũng ta thích lưu trữ gì cũng được. Dữ liệu vẫn được quản lý, thêm sửa xóa độc lập với html template. Khi build ra html sẽ được các trang có đầy đủ data mong muốn.
SSG sử dụng Markdown làm ngôn ngữ soạn thảo bài viết
SSG tạo ra web tĩnh, chỉ có html, css, javascript nên thời gian load cực nhanh.

Lựa chọn Hugo để bắt đầu tạo website tĩnh cho mình

1. Cài đặt Hugo cho Windows
Đầu tiên bạn download file zip mới nhất của Hugo ở địa chỉ này https://github.com/gohugoio/hugo/releases/
Sau khi download về, bạn giải nén sẽ được một thư mục gồm 3 file trong đó có file hugo.exe. File này không dùng để click đúp vào mà các bạn phải làm như sau:

  • BƯớc 1: Bạn tạo thư mục ở hugo ở ổ C.
  • Bước 2: Bạn copy tất cả các file trong file zip hugo bạn vừa giải nén ra bỏ vào thư mục hugo ở ổ C này.
  • Bước 3: Bạn tạo biến môi trường cho hugo theo tương tự link hướng dẫn dưới đây, nhớ thay C:\xampp\vendor\bin trong bước 4 của hướng dẫn bằng C:\hugo

Bây giờ các bạn kiểm tra xem hugo đã được cài đặt chưa bằng lệnh hugo version trong cmd.

2. Tạo trang web bằng Hugo
Đầu tiên, bạn tạo khung project với hugo command trong cmd như sau:

hugo new site [tên website]

Lúc này, trang web của chúng ta đang là trống, chỉ có khung sườn thôi.

5. Đưa trang web lên mạng Chúng ta không cần mua host gì cả, mà hãy đưa code lên github. Nếu bạn chưa có tài khoản Github thì bạn có thể tạo được tài khoản free rất dễ dàng và không giới hạn số code web mà bạn đưa lên. Trang Github có một dịch vụ miễn phí đó là hosting được file html tĩnh. Dịch vụ đó gọi là Github Pages. Bạn đưa code html tĩnh lên Github là bạn có thể hiển thị html đó dưới dạng website thông thường rồi.
> Chú ý: Mình nói html tĩnh ở đây là code html đã được gen bởi Hugo. Như vậy bạn Github mới có thể hiển thị dạng website được. Còn nếu bạn muốn public code bạn đã viết thì cũng có thể upload lên Github bthg nhưng nó sẽ không chuyển đổi thành website cho bạn được.
